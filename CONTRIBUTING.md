# Contributing

Thanks for your interest in improving this benchmark! This is a small project, so the process is lightweight.

## Ways to help

- **Report a problem** — open an issue if a number looks wrong, a step in the README doesn't work, or the methodology could be more sound.
- **Add a fixture** — another real-world project (with a clear MIT/Apache license) or a new synthetic shape.
- **Improve the harness or dashboard** — the benchmark scripts and the `site/` dashboard.

## Local setup

Follow the [Reproduce locally](./README.md#reproduce-locally) section of the README. That gets you both compilers, the fixtures, and a full local run.

## Making a change

1. Fork the repo and create a branch.
2. Make your change and verify it locally:
   - Scripts: `bash -n scripts/<file>.sh` for syntax, then run the pipeline end to end.
   - Dashboard: `cd site && npm run build` must pass (it runs `tsc` + `vite build`).
3. Open a pull request describing **what** you changed and **why**. If it affects the numbers or methodology, say how.

## Things to know

- **Don't hand-edit `results/`.** Those files are generated by `scripts/run-bench.sh` + `scripts/collect-results.ts` and are overwritten by CI on every scheduled run. Change the scripts, not the output.
- **Both compilers are invoked via `npm run ts6` / `npm run ts7`, never `npx tsc`.** See the README for why (`tsc` binary-name collision).
- **The `results/summary.json` schema is the dashboard's contract.** If you change it, update both `scripts/collect-results.ts` (the schema comment lives there) and `site/src/main.ts`.
- **Keep it dependency-light.** The dashboard is intentionally plain TypeScript with no UI framework; please keep new dependencies to a minimum.
- **This repository is public.** Don't commit anything you wouldn't want the world to read.
